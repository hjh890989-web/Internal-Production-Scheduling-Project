### 핵심 로직

#### A. `NotificationService.java` — 진입점

```java
@Service @RequiredArgsConstructor
public class NotificationService {

    private final WebSocketNotificationPublisher wsPublisher;
    private final KakaoTalkClient kakaoClient;
    private final NotificationRepository repo;       // TK-03-3-2

    /**
     * Critical → 인앱 + 카톡
     * Normal → 인앱만
     */
    public void notify(OrderChangeEntity change) {
        Severity sev = Severity.valueOf(change.getSeverity());

        // 1. 인앱 알림 (모든 변경) — WebSocket
        var notif = createNotification(change, NotificationChannel.IN_APP);
        repo.save(notif);
        wsPublisher.publish(notif);

        // 2. Critical만 카카오톡 백업
        if (sev == Severity.CRITICAL) {
            var kakao = createNotification(change, NotificationChannel.KAKAOTALK);
            repo.save(kakao);
            kakaoClient.send(kakao).subscribe(
                resp -> handleSuccess(kakao, resp),
                err -> handleFailure(kakao, err)
            );
        }
    }
}
```

#### B. `KakaoTalkClient.java` — 외부 호출

```java
@Component @RequiredArgsConstructor
public class KakaoTalkClient {

    private final WebClient kakaoWebClient;         // pre-configured
    private final NotificationConfig config;

    /**
     * Critical 변경 알림 → 영향 사용자(P3 압출반장 등) 카카오톡 송신
     */
    @CircuitBreaker(name = "kakaotalk", fallbackMethod = "fallbackSend")
    @Retry(name = "kakaotalk")
    public Mono<KakaoResponse> send(Notification notification) {
        var payload = buildPayload(notification);
        return kakaoWebClient.post()
            .uri(config.getKakao().getWebhookUrl())
            .bodyValue(payload)
            .retrieve()
            .bodyToMono(KakaoResponse.class)
            .timeout(Duration.ofSeconds(5));
    }

    public Mono<KakaoResponse> fallbackSend(Notification n, Throwable cause) {
        log.warn("Kakao send failed for notification {} ({}) — will retry async",
            n.getNotificationId(), cause.getMessage());
        return Mono.just(KakaoResponse.failed(cause.getMessage()));
    }

    private KakaoPayload buildPayload(Notification n) {
        return new KakaoPayload(
            n.getTargetRole(),                      // "PLANNER" 또는 사용자 ID
            buildMessage(n),                        // 한국어 텍스트
            Map.of("notificationId", n.getNotificationId().toString())
        );
    }

    private String buildMessage(Notification n) {
        // 한국어 BizMessage 템플릿 (REQ-NF-USA-003)
        return """
            [🔴 Critical 수주 변경 알림]
            품번: %s
            납기일: %s
            변경 사항: %s
            확인: %s
            """.formatted(
                n.getHoseId(), n.getDeliveryDate(),
                n.getChangeSummary(),
                config.getAppUrl() + "/orders/" + n.getNotificationId());
    }
}
```

#### C. `WebSocketNotificationPublisher.java`

```java
@Component @RequiredArgsConstructor
public class WebSocketNotificationPublisher {

    private final SimpMessagingTemplate stomp;

    public void publish(Notification n) {
        // /topic/notifications/{role} 채널로 PUSH
        stomp.convertAndSend(
            "/topic/notifications/" + n.getTargetRole(),
            n
        );
    }
}
```

#### D. `application.yaml`

```yaml
scheduling:
  notification:
    app-url: https://schedule.intranet
    kakao:
      enabled: true
      webhook-url: https://kapi.kakao.com/v2/api/talk/memo/default/send
      bot-token: ${KAKAO_BOT_TOKEN}
      timeout-seconds: 5

resilience4j:
  circuitbreaker:
    instances:
      kakaotalk:
        sliding-window-size: 100
        failure-rate-threshold: 50
        wait-duration-in-open-state: 30s
  retry:
    instances:
      kakaotalk:
        max-attempts: 3
        wait-duration: 1s
        exponential-backoff-multiplier: 2
```

## :test_tube: Acceptance Criteria

**검증**: T-I + I

- [ ] **Critical → 인앱 + 카톡 양쪽 발송** (NORMAL은 인앱만)

- [ ] **Circuit Breaker** — 카카오 API 50% 실패율 시 Open → 30초 후 Half-Open
- [ ] **Fallback** — 카톡 실패해도 인앱 정상 (장애 격리)
- [ ] **한국어 메시지** 100% (BizMessage 템플릿)
