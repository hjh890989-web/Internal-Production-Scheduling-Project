### 핵심 로직

#### A. Trivy 설정 (`trivy-config.yaml`)

```yaml
# infrastructure/ci/trivy/trivy-config.yaml
severity: HIGH,CRITICAL          # MEDIUM 이하 보고만 (빌드 실패 안 함)
exit-code: 1                     # HIGH/CRITICAL 발견 시 비정상 종료
ignore-unfixed: true             # 패치 없는 CVE는 제외 (실용)
no-progress: true
format: json

scanners:
  - vuln                         # CVE 스캔
  - misconfig                    # IaC misconfiguration
  - secret                       # 하드코딩 secrets 탐지

vulnerability:
  type:
    - os
    - library

cache-dir: /workspace/.trivy-cache
```

#### B. `.trivyignore` — false positive 관리

```
# infrastructure/ci/trivy/.trivyignore
# 형식: CVE-YYYY-NNNN [만료일]
# 알려진 false positive 또는 운영 정책상 수용
# 예시:
# CVE-2023-1234 2026-12-31   # 사유: 패치 예정, 임시 허용
```

#### C. Shared Library 공통 함수 (`vars/trivyScan.groovy`)

```groovy
// infrastructure/jenkins/shared-library/vars/trivyScan.groovy
def call(Map config = [:]) {
    def imageTag    = config.imageTag    ?: error("imageTag required")
    def projectDir  = config.projectDir  ?: '.'
    def reportName  = config.reportName  ?: 'trivy-report'

    sh """
        # 이미지 스캔
        docker run --rm \
            -v /var/run/docker.sock:/var/run/docker.sock \
            -v \$(pwd)/${projectDir}:/workspace \
            -v ${env.WORKSPACE}/.trivy-cache:/root/.cache/trivy \
            aquasec/trivy:latest \
            image \
            --config /workspace/infrastructure/ci/trivy/trivy-config.yaml \
            --ignorefile /workspace/infrastructure/ci/trivy/.trivyignore \
            --output ${reportName}-image.json \
            ${imageTag}

        # 의존성 (Gradle·npm) 스캔
        docker run --rm \
            -v \$(pwd)/${projectDir}:/workspace \
            aquasec/trivy:latest \
            fs \
            --config /workspace/infrastructure/ci/trivy/trivy-config.yaml \
            --output ${reportName}-deps.json \
            /workspace

        # IaC (Dockerfile·docker-compose·k8s YAML) 스캔
        docker run --rm \
            -v \$(pwd)/${projectDir}:/workspace \
            aquasec/trivy:latest \
            config \
            --config /workspace/infrastructure/ci/trivy/trivy-config.yaml \
            --output ${reportName}-iac.json \
            /workspace
    """

    archiveArtifacts artifacts: "${reportName}-*.json", allowEmptyArchive: false
}
```

#### D. Jenkinsfile에 단계 추가

```groovy
// Jenkinsfile.backend (Build Docker Image 후, Push 전)
stage('Security Scan (Trivy)') {
    steps {
        trivyScan(
            imageTag: env.IMAGE_TAG,
            projectDir: 'backend',
            reportName: 'trivy-backend'
        )
    }
    post {
        always {
            // JSON → HTML 변환 (선택)
            sh 'trivy-report-converter trivy-backend-*.json > trivy-backend.html || true'
            publishHTML([reportDir: '.', reportFiles: 'trivy-backend.html', reportName: 'Trivy Scan'])
        }
    }
}
```

## :test_tube: Acceptance Criteria

**1차 검증 방법**: T-I (Integration test) + I (Inspection)

### 측정 가능 기준
- [ ] Trivy 컨테이너 정상 실행 — `docker run --rm aquasec/trivy:latest --version` SUCCESS
- [ ] Backend Docker 이미지 스캔 SUCCESS — HIGH/CRITICAL 0건 시 exit 0
- [ ] 의도적 취약 의존성 추가 시 (예: `org.apache.commons:commons-collections:3.2.1` — 알려진 CVE) **빌드 FAILED**
- [ ] Dockerfile misconfig 스캔 — `USER root` 등 위험 패턴 보고
- [ ] 하드코딩 secret 탐지 — `password: 'plain_text'` 발견 시 보고
- [ ] HTML/JSON 리포트 publish 정상
- [ ] `.trivyignore` 패턴 적용 — 명시된 CVE는 빌드 통과
- [ ] Frontend `package.json` 의존성도 동일 스캔 (npm CVE)

### 회귀 시나리오 (T-I)
- **케이스 1 (정상)**: 깨끗한 코드 → Trivy SUCCESS → 빌드 다음 단계 진행
- **케이스 2 (취약점)**: 의도적 취약 의존성 추가 → 빌드 FAILED + 리포트 archive
- **케이스 3 (false positive)**: `.trivyignore`에 CVE 추가 → 해당 CVE 제외 후 SUCCESS
