#### C. Shared Library 공통 알림 (`vars/notifySlack.groovy`)

```groovy
// infrastructure/jenkins/shared-library/vars/notifySlack.groovy
// 사용: notifySlack(channel: '#scheduling-builds', status: 'SUCCESS', extra: '...')
def call(Map config) {
    def channel = config.channel ?: '#scheduling-builds'
    def status  = config.status  ?: currentBuild.currentResult  // SUCCESS·FAILURE·UNSTABLE·ABORTED
    def extra   = config.extra   ?: ''

    def colorMap = [
        'SUCCESS':  'good',     // 녹색
        'UNSTABLE': 'warning',  // 노랑
        'FAILURE':  'danger',   // 빨강
        'ABORTED':  '#cccccc',  // 회색
    ]
    def emojiMap = [
        'SUCCESS':  ':white_check_mark:',
        'UNSTABLE': ':warning:',
        'FAILURE':  ':x:',
        'ABORTED':  ':octagonal_sign:',
    ]

    def color = colorMap[status] ?: '#cccccc'
    def emoji = emojiMap[status] ?: ':grey_question:'
    def author = env.GIT_AUTHOR ?: 'unknown'
    def commitMsg = (env.GIT_MESSAGE ?: '').take(80)
    def jobName = env.JOB_NAME ?: 'unknown'
    def buildNum = env.BUILD_NUMBER ?: '?'
    def buildUrl = env.BUILD_URL ?: '#'
    def branch = env.BRANCH_NAME ?: 'unknown'
    def duration = currentBuild.durationString.replace(' and counting', '')

    def message = """\
${emoji} *${jobName}* #${buildNum} — ${status}
• 브랜치: `${branch}`
• 커밋: ${author} — ${commitMsg}
• 소요: ${duration}
• <${buildUrl}|빌드 결과 보기>
${extra ? '• ' + extra : ''}\
""".stripIndent()

    slackSend(channel: channel, color: color, message: message)
}
```

#### D. Jenkinsfile post 블록 통합

```groovy
// Jenkinsfile.backend·frontend post 블록 갱신
post {
    success {
        notifySlack(channel: '#scheduling-builds', status: 'SUCCESS')
    }
    failure {
        notifySlack(channel: '#scheduling-builds', status: 'FAILURE')
        // Quality Gate 실패 시 #security 채널에도
        script {
            if (currentBuild.description?.contains('quality gate')) {
                notifySlack(channel: '#scheduling-security', status: 'FAILURE',
                            extra: 'SonarQube Quality Gate 실패 — 즉시 확인 필요')
            }
        }
    }
    unstable {
        notifySlack(channel: '#scheduling-builds', status: 'UNSTABLE')
    }
    aborted {
        notifySlack(channel: '#scheduling-builds', status: 'ABORTED')
    }
}
```

#### E. Trivy 발견 시 보안 채널 알림 (TK-32-2-1 통합)

```groovy
// trivyScan.groovy 갱신
def call(Map config) {
    // ... (기존 스캔 로직)

    // 스캔 결과 JSON 파싱하여 HIGH/CRITICAL 카운트
    def report = readJSON file: "${config.reportName}-image.json"
    def vulnCount = report.Results?.collect { it.Vulnerabilities?.size() ?: 0 }?.sum() ?: 0

    if (vulnCount > 0) {
        notifySlack(
            channel: '#scheduling-security',
            status: 'FAILURE',
            extra: "Trivy 발견: HIGH/CRITICAL ${vulnCount}건 — 이미지: ${config.imageTag}"
        )
        error("Trivy 취약점 ${vulnCount}건 — 빌드 중단")
    }
}
```

## :test_tube: Acceptance Criteria

**1차 검증 방법**: T-I (Integration test) + D (Demonstration)

### 측정 가능 기준
- [ ] Slack Bot Token CASC credentials 등록 + Jenkins UI에서 인식
- [ ] 빌드 SUCCESS 시 `#scheduling-builds` 알림 도달 — **≤60초 (REQ-NF-OPS-003)**
- [ ] 빌드 FAILURE 시 동일 채널 알림 + 적색 표시
- [ ] **SonarQube Quality Gate FAILED 시 `#scheduling-security` 추가 알림**
- [ ] **Trivy HIGH/CRITICAL 발견 시 `#scheduling-security` 알림**
- [ ] PROD 배포 시작·완료·실패 시 `#scheduling-deploys` 알림
- [ ] 알림 메시지에 포함: emoji·브랜치·커밋 author·메시지·소요·빌드 URL
- [ ] Slack 채널 4개 정의 + 구독자 정책 `slack_channels.md` 문서화
- [ ] 알림 미도달 시 자동 재시도 (Slack plugin 기본 동작) + Jenkins 로그에 실패 기록

### 회귀 시나리오 (T-I)
- **케이스 1 (정상 빌드)**: 빌드 SUCCESS → `#builds` 채널 60초 내 녹색 메시지
- **케이스 2 (Quality Gate)**: SonarQube 실패 → `#builds` 적색 + `#security` 추가 메시지
- **케이스 3 (Trivy 위반)**: 의도적 취약 의존성 → `#security` 메시지 + 빌드 중단
- **케이스 4 (배포)**: main 브랜치 SUCCESS → `#deploys` "배포 시작" → "배포 완료" 2건
