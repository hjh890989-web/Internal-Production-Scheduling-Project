#### C. `FolderWatcherService.java` — WatchService 실시간

```java
package com.scheduling.order.watcher;

import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.context.ApplicationEventPublisher;
import org.springframework.stereotype.Service;

import java.io.IOException;
import java.nio.file.*;
import java.time.Instant;

import static java.nio.file.StandardWatchEventKinds.*;

@Slf4j
@Service
@RequiredArgsConstructor
public class FolderWatcherService {

    private final WatcherConfig config;
    private final ApplicationEventPublisher publisher;

    private WatchService watchService;
    private Thread watcherThread;
    private volatile boolean running = false;

    @PostConstruct
    public void start() {
        if (!config.isEnabled()) {
            log.info("Folder watcher disabled (scheduling.watcher.enabled=false)");
            return;
        }

        try {
            Files.createDirectories(config.getWatchFolder());
            watchService = FileSystems.getDefault().newWatchService();
            config.getWatchFolder().register(watchService,
                ENTRY_CREATE, ENTRY_MODIFY);

            running = true;
            watcherThread = new Thread(this::runLoop, "folder-watcher");
            watcherThread.setDaemon(true);
            watcherThread.start();

            log.info("Folder watcher started on {}", config.getWatchFolder());
        } catch (IOException e) {
            log.error("Failed to start folder watcher — falling back to @Scheduled poller only", e);
            // 장애 격리 — watcher 시작 실패가 다른 서비스 영향 없음
        }
    }

    private void runLoop() {
        while (running) {
            WatchKey key;
            try {
                key = watchService.take();
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            } catch (ClosedWatchServiceException e) {
                break;
            }

            for (WatchEvent<?> event : key.pollEvents()) {
                WatchEvent.Kind<?> kind = event.kind();
                if (kind == OVERFLOW) continue;

                Path filename = (Path) event.context();
                Path full = config.getWatchFolder().resolve(filename);

                // 패턴 매칭 (*.xlsx)
                if (!matchesPattern(filename)) continue;

                try {
                    long size = Files.size(full);
                    publisher.publishEvent(new FileDiscoveredEvent(
                        full, size, Instant.now(),
                        FileDiscoveredEvent.Source.WATCH_SERVICE
                    ));
                    log.debug("WatchService detected: {} ({} bytes)", full, size);
                } catch (IOException e) {
                    log.warn("Could not stat file {} — will retry via @Scheduled poller", full, e);
                }
            }

            boolean valid = key.reset();
            if (!valid) {
                log.warn("Watch key invalidated — watcher stopping");
                break;
            }
        }
    }

    private boolean matchesPattern(Path filename) {
        PathMatcher matcher = FileSystems.getDefault()
            .getPathMatcher("glob:" + config.getFilePattern());
        return matcher.matches(filename);
    }

    @PreDestroy
    public void stop() {
        running = false;
        if (watchService != null) {
            try { watchService.close(); } catch (IOException ignored) {}
        }
        if (watcherThread != null) {
            watcherThread.interrupt();
        }
        log.info("Folder watcher stopped");
    }
}
```

#### D. `ScheduledFolderPoller.java` — fallback 폴링

```java
package com.scheduling.order.watcher;

import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import net.javacrumbs.shedlock.spring.annotation.SchedulerLock;
import org.springframework.context.ApplicationEventPublisher;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;

import java.io.IOException;
import java.nio.file.*;
import java.nio.file.attribute.BasicFileAttributes;
import java.time.Instant;
import java.util.Set;
import java.util.concurrent.ConcurrentHashMap;

@Slf4j
@Component
@RequiredArgsConstructor
public class ScheduledFolderPoller {

    private final WatcherConfig config;
    private final ApplicationEventPublisher publisher;

    /** WatchService와 중복 발행 방지용 캐시 — Redis로 옮길 수도 있음 (다중 인스턴스 시) */
    private final Set<String> recentlyDiscovered = ConcurrentHashMap.newKeySet();

    /**
     * 60초마다 폴더 스캔.
     * Shedlock으로 다중 인스턴스 환경에서 1개만 실행 (사내 단일 인스턴스 환경이지만 안전 가드)
     */
    @Scheduled(fixedDelayString = "${scheduling.watcher.poll-interval:PT60S}")
    @SchedulerLock(name = "folder-poller", lockAtLeastFor = "PT10S", lockAtMostFor = "PT55S")
    public void poll() {
        if (!config.isEnabled()) return;

        Path watch = config.getWatchFolder();
        if (!Files.isDirectory(watch)) {
            log.warn("Watch folder does not exist: {}", watch);
            return;
        }

        try (var stream = Files.newDirectoryStream(watch, config.getFilePattern())) {
            for (Path file : stream) {
                if (recentlyDiscovered.contains(file.toString())) continue;

                try {
                    BasicFileAttributes attrs = Files.readAttributes(file, BasicFileAttributes.class);
                    if (!attrs.isRegularFile()) continue;

                    publisher.publishEvent(new FileDiscoveredEvent(
                        file, attrs.size(), Instant.now(),
                        FileDiscoveredEvent.Source.SCHEDULED_POLL
                    ));
                    recentlyDiscovered.add(file.toString());

                    // 중복 방지 캐시는 5분 후 자동 정리 (단순 구현 — 향후 Redis TTL로)
                } catch (IOException e) {
                    log.warn("Could not read attributes of {}", file, e);
                }
            }
        } catch (IOException e) {
            log.error("Polling failed for {}", watch, e);
        }
    }

    @Scheduled(fixedDelay = 5 * 60 * 1000)
    public void cleanupCache() {
        recentlyDiscovered.clear();   // 5분마다 캐시 비움 (단순 패턴)
    }
}
```

#### F. `WatcherHealthIndicator.java` — Actuator

```java
package com.scheduling.order.watcher;

import lombok.RequiredArgsConstructor;
import org.springframework.boot.actuate.health.*;
import org.springframework.stereotype.Component;

import java.nio.file.Files;

@Component
@RequiredArgsConstructor
public class WatcherHealthIndicator implements HealthIndicator {

    private final WatcherConfig config;

    @Override
    public Health health() {
        if (!config.isEnabled()) {
            return Health.up().withDetail("status", "disabled").build();
        }
        if (!Files.isDirectory(config.getWatchFolder())) {
            return Health.down()
                .withDetail("status", "folder-missing")
                .withDetail("watchFolder", config.getWatchFolder().toString())
                .build();
        }
        if (!Files.isReadable(config.getWatchFolder())) {
            return Health.down().withDetail("status", "not-readable").build();
        }
        return Health.up()
            .withDetail("status", "active")
            .withDetail("watchFolder", config.getWatchFolder().toString())
            .build();
    }
}
```

## :memo: Implementation Notes

- **기술 힌트**:
  - Java NIO WatchService는 Linux inotify 기반 — Linux/macOS에서 즉시 이벤트, Windows는 일부 지연
  - `WatchService.take()`는 blocking — daemon thread로 격리 필수
  - Spring `@Scheduled` + Shedlock으로 다중 인스턴스 안전성 (사내 단일 인스턴스 환경이라도 코드 일관성)
  - `Files.size()` 호출 시 IOException 가능 — 파일이 쓰기 중이거나 삭제됐을 수 있음 (TK-01-3-3에서 안정화)
  - **장애 격리**: `@PostConstruct`에서 예외 발생해도 Spring context는 정상 (try-catch + 로깅)
- **유사 패턴**:
  - 미래 PDF·CSV 자동 ingest watcher도 동일 패턴
- **표준 참조**:
  - JEP 392 (Sealed Classes, Java 17+) — 향후 `FileDiscoveredEvent.Source` 확장 시 활용
  - Spring Boot Actuator Health: https://docs.spring.io/spring-boot/docs/3.3.x/reference/html/actuator.html#actuator.endpoints.health
- **참고 외부 자료**:
  - "Watching a Directory for Changes" — Oracle Java Tutorials
