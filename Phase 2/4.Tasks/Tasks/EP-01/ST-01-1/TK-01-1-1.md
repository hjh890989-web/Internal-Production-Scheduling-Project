#### B. `ExcelParserService.java` — 스트리밍 파서

```java
package com.scheduling.order.parser;

import com.github.pjfanning.xlsx.StreamingReader;
import org.apache.poi.ss.usermodel.*;
import org.springframework.stereotype.Service;

import java.io.InputStream;
import java.util.*;

@Service
public class ExcelParserService {

    private static final int ROW_CACHE_SIZE = 100;        // 메모리 ≤ 256 MB 보장
    private static final int BUFFER_SIZE = 4096;
    private static final long MAX_FILE_SIZE_BYTES = 20L * 1024 * 1024;  // 20 MB (REQ-FUNC-OC-001)

    public ParsedWorkbook parse(String filename, InputStream is, long fileSize) {
        if (fileSize > MAX_FILE_SIZE_BYTES) {
            throw new ExcelParseException("FILE_TOO_LARGE",
                "Workbook %s exceeds 20MB limit (%d bytes)".formatted(filename, fileSize));
        }

        ParsedWorkbook result = new ParsedWorkbook(filename);

        // SXSSF Streaming Reader — 청크 단위 row 읽기 (메모리 최적)
        try (Workbook workbook = StreamingReader.builder()
                .rowCacheSize(ROW_CACHE_SIZE)
                .bufferSize(BUFFER_SIZE)
                .open(is)) {

            for (Sheet sheet : workbook) {
                result.addSheet(parseSheet(sheet));
            }

        } catch (Exception e) {
            throw new ExcelParseException("PARSE_FAILED",
                "Failed to parse %s: %s".formatted(filename, e.getMessage()), e);
        }

        return result;
    }

    private ParsedSheet parseSheet(Sheet sheet) {
        ParsedSheet ps = new ParsedSheet(sheet.getSheetName());
        int rowIdx = 0;

        for (Row row : sheet) {
            List<String> cells = new ArrayList<>();
            int maxCol = row.getLastCellNum();

            for (int c = 0; c < maxCol; c++) {
                Cell cell = row.getCell(c, Row.MissingCellPolicy.CREATE_NULL_AS_BLANK);
                cells.add(extractCellValue(cell));
            }

            ps.addRow(new ParsedRow(rowIdx++, cells));
        }

        return ps;
    }

    private String extractCellValue(Cell cell) {
        if (cell == null) return "";
        return switch (cell.getCellType()) {
            case STRING -> cell.getStringCellValue();
            case NUMERIC -> DateUtil.isCellDateFormatted(cell)
                ? cell.getDateCellValue().toString()
                : String.valueOf(cell.getNumericCellValue());
            case BOOLEAN -> String.valueOf(cell.getBooleanCellValue());
            case FORMULA -> cell.getCellFormula();   // 수식 자체 보존 (역-Export용)
            case BLANK -> "";
            default -> cell.toString();
        };
    }
}
```

#### C. `ParsedWorkbook.java` — DTO

```java
package com.scheduling.order.parser;

import java.util.*;

public record ParsedWorkbook(
    String filename,
    List<ParsedSheet> sheets
) {
    public ParsedWorkbook(String filename) {
        this(filename, new ArrayList<>());
    }
    public void addSheet(ParsedSheet sheet) { sheets.add(sheet); }
}

public record ParsedSheet(
    String sheetName,
    List<ParsedRow> rows
) {
    public ParsedSheet(String sheetName) {
        this(sheetName, new ArrayList<>());
    }
    public void addRow(ParsedRow row) { rows.add(row); }
}

public record ParsedRow(
    int index,         // 0-based row index
    List<String> cells // 0-based cell values
) {}
```

```java
package com.scheduling.order.parser;

public class ExcelParseException extends RuntimeException {
    private final String code;   // "FILE_TOO_LARGE", "PARSE_FAILED", "UNRECOGNIZED_FORMAT"

    public ExcelParseException(String code, String message) {
        super(message);
        this.code = code;
    }
    public ExcelParseException(String code, String message, Throwable cause) {
        super(message, cause);
        this.code = code;
    }
    public String getCode() { return code; }
}
```

### 측정 가능 기준

- [ ] **10,000 row 워크북** parse 후 JVM 힙 사용 ≤ 256 MB (`-Xmx256m`로 실행 가능)
- [ ] **10,000 row 워크북** parse 응답 p95 ≤ 30초 (TK-01-1-3의 추적 ID 발급과 합쳐 NFR-PER-001 60초 충족)
- [ ] **20MB 초과 파일** → `ExcelParseException("FILE_TOO_LARGE")` 예외 + HTTP 413
- [ ] **손상된 .xlsx 파일** → `ExcelParseException("PARSE_FAILED")` + row-level 에러 정보
- [ ] **병합 셀** 처리 → 첫 셀에 값 보존, 나머지는 빈 문자열
- [ ] **날짜 셀** → ISO-8601 형식 문자열 (KST 적용은 ST-01-2에서)
- [ ] **수식 셀** → 수식 원본 보존 (`=SUM(A1:A10)`) — 역-Export 대비 (REQ-FUNC-OC-013)
- [ ] **빈 셀** → 빈 문자열 (`""`)

## :memo: Implementation Notes

- **기술 힌트**:
  - `excel-streaming-reader` (pjfanning) ≫ POI 자체 SAX API — 더 간단, 메모리 동등
  - `Row.MissingCellPolicy.CREATE_NULL_AS_BLANK`로 빈 셀 일관 처리 (NullPointerException 방지)
  - 수식 셀은 `cell.getCellFormula()`로 원본 보존 — 역-Export(REQ-FUNC-OC-013)에서 그대로 다시 출력
  - `DateUtil.isCellDateFormatted(cell)`로 날짜 자동 인식 (셀 포맷 의존)
  - **메모리 프로파일링**: VisualVM 또는 JFR(Java Flight Recorder)로 힙 사용량 측정
- **유사 패턴**:
  - 향후 PDF·CSV 파서가 필요하면 동일한 `XxxParserService` + `ParsedDocument` 추상화 가능
- **표준 참조**:
  - Apache POI XSSF: https://poi.apache.org/components/spreadsheet/
  - OpenXML SpreadsheetML: ECMA-376
- **참고 외부 자료**:
  - excel-streaming-reader: https://github.com/pjfanning/excel-streaming-reader
  - "How to read large Excel files with Apache POI": https://stackoverflow.com/questions/11118341/large-xlsx-files-with-apache-poi
