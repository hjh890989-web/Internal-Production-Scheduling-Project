### `pg-backup.sh`

```bash
#!/bin/bash
set -e
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/var/backups/postgresql/${TIMESTAMP}"
mkdir -p "${BACKUP_DIR}"

# pg_basebackup with WAL streaming
pg_basebackup -h ${DB_HOST} -U ${PG_BACKUP_USER} -D "${BACKUP_DIR}" \
  --format=tar --gzip --progress --verbose \
  --checkpoint=fast --wal-method=stream

# S3 또는 NAS 업로드
aws s3 cp "${BACKUP_DIR}" "s3://${BACKUP_BUCKET}/postgres/${TIMESTAMP}/" \
  --recursive --storage-class STANDARD_IA

# 30일 retention 적용
find /var/backups/postgresql -maxdepth 1 -type d -mtime +30 -exec rm -rf {} +

# Slack 알림
curl -X POST ${SLACK_WEBHOOK_URL} -d "{\"text\":\"PG backup complete: ${TIMESTAMP}\"}"
```

### `systemd/pg-backup.timer`

```ini
[Unit]
Description=PostgreSQL daily backup at 02:00 KST

[Timer]
OnCalendar=*-*-* 02:00:00 Asia/Seoul
Persistent=true

[Install]
WantedBy=timers.target
```

### `systemd/pg-backup.service`

```ini
[Unit]
Description=PostgreSQL pg_basebackup
After=docker.service

[Service]
Type=oneshot
EnvironmentFile=/etc/scheduling/.env.prod
ExecStart=/usr/local/bin/pg-backup.sh
User=backup
```

## :memo: Implementation Notes

- STANDARD_IA — 백업 cost 절감 (자주 접근 안 함)
- 백업 storage 권한 분리 (backup_user) — root 아닌 backup_user만 작성
- 검증 자동화: 매 백업 후 tar 파일 size 검증 (≥ 100MB)
