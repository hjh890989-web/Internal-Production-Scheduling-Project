### `GreedyRotationAllocator` 수정

```java
@Service @RequiredArgsConstructor
public class GreedyRotationAllocator {

    private final RulePipeline pipeline;
    private final AllocationLedger ledger;
    // ...

    public AllocationResult allocate(AllocationContext ctx) {
        ledger.clear();    // 신규 호출마다 reset

        var schedule = new ArrayList<AllocationRow>();
        var conflicts = new ArrayList<AllocationConflict>();

        // ... 호라이즌 순회 (date / machine / rotation / slot)
        for (LocalDate date : ctx.workingDays()) {
            for (var machine : candidateMachines(...)) {
                for (var rotation : machine.rotations(date)) {
                    for (int slot = 1; slot <= machine.slotCount(); slot++) {
                        var cand = SlotCandidate.of(hose, date, machine, rotation, slot);

                        var result = pipeline.evaluate(cand, ledger);
                        if (!result.passed()) {
                            conflicts.add(AllocationConflict.from(cand, result));
                            continue;
                        }

                        schedule.add(AllocationRow.from(cand));

                        // 핵심: ledger 갱신 — 후속 후보가 IntraDayLockRule로 차단
                        ledger.recordAssignment(cand);
                    }
                }
            }
        }

        return new AllocationResult(schedule, conflicts);
    }
}
```
