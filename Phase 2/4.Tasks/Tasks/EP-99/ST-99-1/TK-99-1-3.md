### 핵심 로직 / 알고리즘

본 Task의 핵심은 **PostgreSQL `master` 스키마의 무결성 검사 SQL 4종**:

#### CHECK-1: K/L 컬럼 값 무결성 (CHECK 제약 우회 탐지)
```sql
-- DDL의 CHECK 제약이 적용되지 않는 경우 (예: 마이그레이션 미실행) 위반 탐지
SELECT hose_id, lp_left_setting, lp_right_setting
FROM master.vc_constraint
WHERE lp_left_setting  NOT IN ('o','x')
   OR lp_right_setting NOT IN ('o','x')
   OR lp_left_setting  IS NULL
   OR lp_right_setting IS NULL;
-- 기대: 0 row (모든 47품번 정합)
```

#### CHECK-2: `VC_HOSE_RULE` 시드 데이터 ↔ DDL CHECK 정합
```sql
-- side_lock CHECK 위반·미정의 machine_pin·max_concurrent_slots 범위 외 탐지
SELECT hose_id, machine_pin, max_concurrent_slots, side_lock, lp_only, rule_source
FROM master.vc_hose_rule
WHERE (side_lock IS NOT NULL AND side_lock NOT IN ('L','R'))
   OR (max_concurrent_slots IS NOT NULL AND max_concurrent_slots NOT BETWEEN 1 AND 8)
   OR (machine_pin IS NOT NULL AND machine_pin NOT IN ('LP-01','LP-02','LP-03','LP-04','IC-01'))
   OR rule_source NOT LIKE 'BR-V%';
-- 기대: 0 row
```

#### CHECK-3: `VC_HOSE_RULE` ↔ `VC_CONSTRAINT` 좌/우 일관성
```sql
-- side_lock='R' (우측 only) 이면 마스터의 lp_left_setting='x' AND lp_right_setting='o' 여야 함
-- side_lock='L' (좌측 only) 이면 lp_left_setting='o' AND lp_right_setting='x' 여야 함
WITH rule_mismatch AS (
    SELECT
        r.hose_id, r.side_lock,
        c.lp_left_setting AS k, c.lp_right_setting AS l,
        CASE
            WHEN r.side_lock = 'R' AND (c.lp_left_setting != 'x' OR c.lp_right_setting != 'o') THEN 'R 룰 위반'
            WHEN r.side_lock = 'L' AND (c.lp_left_setting != 'o' OR c.lp_right_setting != 'x') THEN 'L 룰 위반'
            ELSE NULL
        END AS issue
    FROM master.vc_hose_rule r
    JOIN master.vc_constraint c USING (hose_id)
    WHERE r.side_lock IS NOT NULL
)
SELECT * FROM rule_mismatch WHERE issue IS NOT NULL;
-- 기대: 0 row (28422-2M800·28421-2M800 모두 정합)
```

#### CHECK-4: 시드 ↔ 명세 매트릭스 일치 (BR-V14·V15·V16 직접 검증)
```sql
-- TK-99-1-2의 cross-check 매트릭스를 SQL로 표현
WITH expected (hose_id, exp_pin, exp_max, exp_lock, exp_lp_only) AS (
    VALUES
        ('28422-08HA0', 'LP-01', 1, NULL, TRUE),
        ('28422-2M800', NULL,    2, 'R',  TRUE),
        ('28421-2M800', NULL,    2, 'L',  TRUE)
)
SELECT
    e.hose_id,
    e.exp_pin, r.machine_pin AS act_pin,
    e.exp_max, r.max_concurrent_slots AS act_max,
    e.exp_lock, r.side_lock AS act_lock,
    e.exp_lp_only, r.lp_only AS act_lp_only
FROM expected e
LEFT JOIN master.vc_hose_rule r USING (hose_id)
WHERE r.hose_id IS NULL                                    -- 시드 누락
   OR e.exp_pin IS DISTINCT FROM r.machine_pin             -- machine_pin 불일치
   OR e.exp_max IS DISTINCT FROM r.max_concurrent_slots
   OR e.exp_lock IS DISTINCT FROM r.side_lock
   OR e.exp_lp_only IS DISTINCT FROM r.lp_only;
-- 기대: 0 row (3 품번 모두 시드 정합)
```

END;
$$ LANGUAGE plpgsql;

### 셸 wrapper 예시 (`tools/master_validation/run_integrity_check.sh`):
```bash
#!/usr/bin/env bash
# 사용: ./run_integrity_check.sh <db_url>
DB_URL="${1:-postgres://app_user:***@localhost:5432/scheduling}"
DATE=$(date +%Y-%m-%d)
REPORT="tools/master_validation/reports/vc_master_integrity_${DATE}.md"

echo "# VC 마스터 무결성 검사 리포트 (${DATE})" > "$REPORT"
echo "" >> "$REPORT"

psql "$DB_URL" -At -F'|' -c "SELECT check_name, violation_count, details FROM master.check_vc_integrity();" \
    | awk -F'|' 'BEGIN{print "| Check | 위반 | 상세 |"; print "|---|:---:|---|"} {print "| " $1 " | " $2 " | " $3 " |"}' \
    >> "$REPORT"

# 위반 0건이면 exit 0, 1건 이상이면 exit 1
violation_total=$(psql "$DB_URL" -At -c "SELECT COALESCE(SUM(violation_count), 0) FROM master.check_vc_integrity();")
[[ "$violation_total" -eq 0 ]] && exit 0 || exit 1
```

## :memo: Implementation Notes
- **기술 힌트**:
  - PostgreSQL `IS DISTINCT FROM` 연산자가 NULL-safe 비교 — `=`/`!=`보다 안전 (NULL 비교 시 결과가 NULL이 되는 SQL 표준 회피)
  - `string_agg(... ORDER BY ...)`로 위반 품번 목록을 결정적 순서로 출력 (회귀 테스트 reproducibility)
  - 셸 wrapper에서 `psql -At -F'|'`로 awk-friendly 출력 + exit code 명시
- **유사 패턴**:

- **표준 참조**:
  - ISO/IEC/IEEE 12207 §6.4.10 Maintenance (지속 무결성)
  - PMBOK 7th Quality Management §8 (회귀 검사 자동화)
  - PostgreSQL Docs §38 PL/pgSQL: https://www.postgresql.org/docs/16/plpgsql.html
