**cross-check 스크립트** (요약):
```python
#!/usr/bin/env python3
# tools/master_validation/cross_check_hose_rules.py
import sys, io
sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
import openpyxl
from pathlib import Path

EXPECTED = {
    '28422-08HA0': {'K': None, 'L': None, 'side_lock': None, 'max_slots': 1, 'lp_only': True,  'machine_pin': 'LP-01'},
    '28422-2M800': {'K': 'x',  'L': 'o',  'side_lock': 'R',  'max_slots': 2, 'lp_only': True,  'machine_pin': None},
    '28421-2M800': {'K': 'o',  'L': 'x',  'side_lock': 'L',  'max_slots': 2, 'lp_only': True,  'machine_pin': None},
}

def cross_check():
    wb = openpyxl.load_workbook(
        'Phase 1/2.Raw Materials/Vulcanization/성형공정_제약조건.xlsx',
        data_only=True)
    ws = wb['Sheet1 (2)']
    violations = []
    for r in range(3, ws.max_row + 1):
        hose = ws.cell(row=r, column=1).value
        if hose in EXPECTED:
            exp = EXPECTED[hose]
            k = ws.cell(row=r, column=11).value
            l = ws.cell(row=r, column=12).value
            if exp['K'] is not None and k != exp['K']:
                violations.append((hose, f'K 기대 {exp["K"]}, 실제 {k}'))
            if exp['L'] is not None and l != exp['L']:
                violations.append((hose, f'L 기대 {exp["L"]}, 실제 {l}'))
    return violations

if __name__ == '__main__':
    v = cross_check()
    if v:
        for hose, msg in v:
            print(f'⚠️ {hose}: {msg}')
        sys.exit(1)
    print('✅ 3 품번 cross-check 통과')
    sys.exit(0)
```

## :memo: Implementation Notes
- **기술 힌트**:
  - PostgreSQL `ON CONFLICT (...) DO UPDATE` 패턴으로 idempotent 시드 — Flyway 마이그레이션 재실행에도 안전
  - `side_lock IN ('L','R',NULL)` CHECK 제약은 SQL에서 `IS NULL`로 NULL을 명시적 허용 필요 (`side_lock IS NULL OR side_lock IN ('L','R')`)
- **유사 패턴**: 향후 다른 품번 특수 제약 추가 시 동일 cross-check 패턴 + 시드 row 추가. 룰셋 YAML 외부화 검토.
- **표준 참조**: ISO/IEC/IEEE 12207 §6.4.10 Maintenance. PMBOK 7th Quality Management (cross-check via dual data source).
- **참고 외부 자료**:
  - Flyway docs: https://flywaydb.org/documentation/concepts/migrations
  - PostgreSQL CHECK constraints: https://www.postgresql.org/docs/16/ddl-constraints.html
