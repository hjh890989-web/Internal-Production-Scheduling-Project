### 핵심 로직 / 알고리즘

#### A. `common` 모듈 — 모든 모듈에 공개

```java
// common/src/main/java/com/scheduling/common/package-info.java
@org.springframework.modulith.NamedInterface("api")
package com.scheduling.common;
```

**`common` 모듈은 모든 다른 모듈이 의존 가능**. 단, `common`은 다른 어떤 모듈에도 의존 금지 (순환 금지 — SAD §4.1).

```java
// common/build.gradle.kts
dependencies {
    implementation(libs.spring.modulith.starter.core)
    // 다른 subproject 의존성 0건 (순환 금지 보장)
}
```

#### B. 각 도메인 모듈 — public API 패키지 명시

```java
// order/src/main/java/com/scheduling/order/package-info.java
@org.springframework.modulith.ApplicationModule(
    displayName = "Order Consolidation",
    allowedDependencies = { "common" }
)
package com.scheduling.order;
```

```java
// order/src/main/java/com/scheduling/order/api/package-info.java
@org.springframework.modulith.NamedInterface("api")
package com.scheduling.order.api;
```

```java
// order/src/main/java/com/scheduling/order/events/package-info.java
@org.springframework.modulith.NamedInterface("events")
package com.scheduling.order.events;
```

**내부 패키지** (`domain`·`internal`)는 `package-info.java` 없음 → 외부에서 직접 import 불가 (Spring Modulith가 빌드 시 차단).

#### C. 모듈별 의존성 매트릭스 (SAD §4 다이어그램 인용)

| 모듈 | 허용 의존 (`allowedDependencies`) | 비고 |
|---|---|---|
| `common` | (없음) | 모든 모듈이 의존 가능, 본인은 의존 0 |
| `order` | `common`, `master::api`, `audit::events` | 마스터 조회 + audit 이벤트 발행 |
| `vc` | `common`, `master::api`, `audit::events`, `order::events` | 수주 변경 이벤트 구독 |
| `ex` | `common`, `master::api`, `audit::events`, `vc::events` | vc.changed 이벤트 구독 (EX-013) |
| `master` | `common`, `audit::events` | 마스터 변경 시 audit 발행 |
| `audit` | `common` | 모든 모듈의 audit 이벤트 수신 (단방향) |
| `notify` | `common`, `vc::events`, `ex::events`, `order::events` | 변경 이벤트 → 알림 발송 |

> **핵심 원칙**: 모듈은 **이벤트 발행**만 직접, **이벤트 구독**은 자동 (`@ApplicationModuleListener`). 직접 메서드 호출 시 컴파일·빌드 단계에서 Spring Modulith 차단.

#### D. 이벤트 정의 예시 (`order::events`)

```java
// order/src/main/java/com/scheduling/order/events/OrderChangedEvent.java
package com.scheduling.order.events;

import com.scheduling.common.enums.ChangeSeverity;
import java.time.Instant;
import java.util.UUID;

public record OrderChangedEvent(
    UUID orderId,
    String hoseId,
    Instant occurredAt,
    ChangeSeverity severity   // CRITICAL | NORMAL — BR-O02
) {}
```

```java
// vc/src/main/java/com/scheduling/vc/internal/OrderChangedListener.java
package com.scheduling.vc.internal;

import com.scheduling.order.events.OrderChangedEvent;
import org.springframework.modulith.events.ApplicationModuleListener;
import org.springframework.stereotype.Component;

@Component
class OrderChangedListener {
    @ApplicationModuleListener   // = @TransactionalEventListener + @Async
    void on(OrderChangedEvent event) {
        // 영향 받은 VC_SCHEDULE row 식별 + partial replan (REQ-FUNC-VC-015)
    }
}
```

#### E. `@Modulithic` 적용

```java
// app/src/main/java/com/scheduling/SchedulingApplication.java
package com.scheduling;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.modulith.Modulithic;

@Modulithic(
    systemName = "Internal Production Scheduling System",
    sharedModules = "common"   // common은 모든 모듈에서 공유
)
@SpringBootApplication
public class SchedulingApplication {
    public static void main(String[] args) {
        SpringApplication.run(SchedulingApplication.class, args);
    }
}
```

#### F. Modulith 문서 자동 생성 (선택, 권장)

```java
// app/src/test/java/com/scheduling/ModulithDocumentationTest.java
package com.scheduling;

import org.junit.jupiter.api.Test;
import org.springframework.modulith.core.ApplicationModules;
import org.springframework.modulith.docs.Documenter;

class ModulithDocumentationTest {
    static final ApplicationModules MODULES = ApplicationModules.of(SchedulingApplication.class);

    @Test
    void writeDocumentation() {
        new Documenter(MODULES)
            .writeDocumentation()
            .writeIndividualModulesAsPlantUml();
    }
}
```

→ `app/build/spring-modulith-docs/` 에 7 모듈 PlantUML 다이어그램 + 매니페스트 자동 생성. SAD §4 다이어그램과 1:1 매핑 확인 가능.

## :memo: Implementation Notes
- **기술 힌트**:
  - `@ApplicationModuleListener` = `@TransactionalEventListener(AFTER_COMMIT)` + `@Async` 결합 — 트랜잭션 커밋 후 비동기 처리 (성능 + 신뢰성)
  - `sharedModules = "common"` → `common`은 verify 시 의존성 체크에서 면제
  - `NamedInterface("events")`는 모듈의 **공식 발행 이벤트 패키지** 표시 — 다른 모듈이 이 패키지만 import 가능
- **유사 패턴**:
  - 향후 모듈 추가 시 (예: Phase 2 MRP `mrp` 모듈) 동일 패턴 적용 — `package-info.java` + `allowedDependencies`

  - Spring Modulith 1.2 Sample: https://github.com/spring-projects/spring-modulith/tree/main/spring-modulith-examples
