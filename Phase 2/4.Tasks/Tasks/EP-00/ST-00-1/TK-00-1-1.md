### 핵심 로직 / 알고리즘

#### A. Docker Compose 서비스 정의

```yaml
# infrastructure/docker-compose.yml (PG 부분만)
services:
  postgres:
    image: postgres:16-alpine
    container_name: scheduling-postgres
    environment:
      POSTGRES_DB: ${POSTGRES_DB:-scheduling}
      POSTGRES_USER: ${POSTGRES_USER:-app_user}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:?password 필수}
      TZ: Asia/Seoul
      PGTZ: Asia/Seoul
    ports:
      - "127.0.0.1:5432:5432"   # DEV: localhost only
    volumes:
      - postgres-data:/var/lib/postgresql/data
      - ./postgres/init:/docker-entrypoint-initdb.d:ro
      - ./postgres/postgresql.conf:/etc/postgresql/postgresql.conf:ro
    command: postgres -c config_file=/etc/postgresql/postgresql.conf
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-app_user} -d ${POSTGRES_DB:-scheduling}"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s
    restart: unless-stopped
    networks:
      - scheduling-net

volumes:
  postgres-data:
    driver: local

networks:
  scheduling-net:
    driver: bridge
```

#### B. 초기 스키마 생성 (`01_create_schemas.sql`)

```sql
-- 3 스키마 + role 분리 (SAD §6.1.1)
CREATE SCHEMA IF NOT EXISTS app;
CREATE SCHEMA IF NOT EXISTS audit;
CREATE SCHEMA IF NOT EXISTS master;

-- Role 분리 (NFR-SEC-004 audit 불변성)
CREATE ROLE auditor      NOLOGIN;
CREATE ROLE master_admin NOLOGIN;

COMMENT ON SCHEMA app    IS '운영 데이터 (PRODUCT·ORDER·VC_SCHEDULE·EX_SCHEDULE)';
COMMENT ON SCHEMA audit  IS '감사 로그 — INSERT only, ≥3년 보존 (NFR-SEC-004)';
COMMENT ON SCHEMA master IS '마스터 데이터 (VC_CONSTRAINT·EX_CONSTRAINT·VC_HOSE_RULE 등)';

-- KST 기본 timezone (BR-X04, REQ-FUNC-CO-007)
ALTER DATABASE scheduling SET timezone TO 'Asia/Seoul';
```

#### C. 권한 부여 (`02_grant_permissions.sql`)

```sql
-- 애플리케이션 user (app_user) — app/master SELECT·INSERT·UPDATE·DELETE, audit INSERT만
GRANT USAGE ON SCHEMA app, master, audit TO app_user;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA app TO app_user;
GRANT SELECT ON ALL TABLES IN SCHEMA master TO app_user;
GRANT INSERT ON ALL TABLES IN SCHEMA audit TO app_user;

-- auditor — audit SELECT만
GRANT USAGE ON SCHEMA audit TO auditor;
GRANT SELECT ON ALL TABLES IN SCHEMA audit TO auditor;

-- master_admin — master UPDATE (BR-X05 dual-review 후 트리거에서 적용)
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA master TO master_admin;

-- 신규 테이블 생성 시 기본 권한 적용
ALTER DEFAULT PRIVILEGES IN SCHEMA app    GRANT ALL          ON TABLES TO app_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA master GRANT SELECT       ON TABLES TO app_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA audit  GRANT INSERT       ON TABLES TO app_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA audit  GRANT SELECT       ON TABLES TO auditor;
ALTER DEFAULT PRIVILEGES IN SCHEMA master GRANT ALL          ON TABLES TO master_admin;

-- audit UPDATE·DELETE 차단 (REQ-FUNC-CO-005, REQ-NF-SEC-004)
-- ※ ALTER DEFAULT PRIVILEGES는 신규 테이블만 영향 — 실제 audit 트리거는 EP-11 ST-11-2에서 적용
```

## :test_tube: Acceptance Criteria

**1차 검증 방법** (SRS §4.1.6 카탈로그):
- **1차**: T-I (Integration test — 컨테이너 부팅 + psql 접속 + 스키마 존재 확인)
- **2차**: I (Inspection — postgresql.conf·SQL·docker-compose.yml 설정 검토)

### 측정 가능 기준
- [ ] `docker compose up postgres -d` 한 번에 컨테이너 부팅 성공
- [ ] healthcheck 통과 (`docker compose ps`에서 `(healthy)` 표시) ≤30초 이내
- [ ] `psql -h localhost -p 5432 -U app_user -d scheduling -c "\dn"` → 3 스키마(`app`·`audit`·`master`) 모두 존재
- [ ] `psql ... -c "SELECT current_setting('timezone')"` → `Asia/Seoul` 반환
- [ ] `docker compose down` → `up` 재기동 시 스키마·데이터 영속 보존 (volume 정상)
- [ ] app_user는 audit 스키마에 INSERT만 가능, SELECT·UPDATE·DELETE 시 403/permission denied
- [ ] auditor role은 audit SELECT만 가능
- [ ] `POSTGRES_PASSWORD` 환경 변수 미설정 시 컨테이너 부팅 실패 (`:?password 필수` 강제)

### 회귀 시나리오 (T-I)
- **케이스 1 (정상 부팅)**: `.env` 정상 → 30초 내 healthy
- **케이스 2 (패스워드 미설정)**: `POSTGRES_PASSWORD` 제거 → 부팅 실패 + 명확한 에러
- **케이스 3 (재기동 영속성)**: 임시 테이블 생성 → down → up → 테이블 존재 확인
- **케이스 4 (audit 권한)**: app_user로 audit 테이블 UPDATE 시도 → permission denied
