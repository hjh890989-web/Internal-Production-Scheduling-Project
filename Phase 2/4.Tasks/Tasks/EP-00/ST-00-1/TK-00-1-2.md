### 핵심 로직 / 알고리즘

#### A. Docker Compose 서비스 정의

```yaml
# infrastructure/docker-compose.yml (redis 부분 추가)
services:
  # ... postgres 위에 정의됨 (TK-00-1-1)

  redis:
    image: redis:7-alpine
    container_name: scheduling-redis
    command:
      - redis-server
      - /etc/redis/redis.conf
      - --requirepass
      - ${REDIS_PASSWORD:?password 필수}
    ports:
      - "127.0.0.1:6379:6379"   # DEV: localhost only
    volumes:
      - redis-data:/data
      - ./redis/redis.conf:/etc/redis/redis.conf:ro
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD}", "ping"]
      interval: 10s
      timeout: 3s
      retries: 5
      start_period: 10s
    restart: unless-stopped
    networks:
      - scheduling-net

volumes:
  postgres-data:
  redis-data:    # 추가
    driver: local
```

#### B. `redis.conf` 핵심 설정

```conf
# infrastructure/redis/redis.conf

# Network
bind 0.0.0.0
protected-mode yes
port 6379
timeout 300                      # idle 5분 후 자동 종료
tcp-keepalive 60

# AOF 영속성 (NFR-REL-002, RPO 1초)
appendonly yes
appendfsync everysec             # 1초 단위 fsync — 성능·내구성 균형
no-appendfsync-on-rewrite no
auto-aof-rewrite-percentage 100
auto-aof-rewrite-min-size 64mb

# RDB 보조 (시간 단위 스냅샷)
save 3600 1                      # 1시간에 1번 이상 쓰기 시
save 300 100
save 60 10000

# Memory (캐시 정책)
maxmemory 1gb                    # 잉여 서버 8 GB의 1/8 (NFR-COS-001)
maxmemory-policy allkeys-lru     # 메모리 부족 시 LRU eviction

# Pub/Sub (NFR-PER-004 WebSocket 백업 경로)
notify-keyspace-events KEA       # keyspace events 전체 알림 (cache invalidation 용)

# Slowlog (NFR-OPS-001 로깅)
slowlog-log-slower-than 10000    # 10ms 초과 명령
slowlog-max-len 128

# Logging
loglevel notice
logfile ""                       # stderr → docker logs

# Security
# requirepass는 docker-compose command line에서 --requirepass로 주입
```

#### C. 운영 가이드 (`docs/infrastructure/redis_setup.md`)

운영자가 알아야 할 것:

## :test_tube: Acceptance Criteria

**1차 검증 방법** (SRS §4.1.6 카탈로그):
- **1차**: T-I (Integration test — 컨테이너 부팅 + redis-cli 접속 + 명령 실행)
- **2차**: I (Inspection — redis.conf·docker-compose.yml 설정 검토)

### 측정 가능 기준
- [ ] `docker compose up redis -d` 한 번에 컨테이너 부팅 성공
- [ ] healthcheck 통과 (`docker compose ps`에서 `(healthy)`) ≤10초 이내
- [ ] `redis-cli -h localhost -p 6379 -a <password> PING` → `PONG`
- [ ] `redis-cli ... SET foo bar` 후 `GET foo` → `bar` (R/W 정상)
- [ ] `docker compose down` → `up` 재기동 시 AOF에서 데이터 복원 (`GET foo` → `bar` 유지)
- [ ] `redis-cli ... CONFIG GET maxmemory` → `1073741824` (1 GB)
- [ ] `redis-cli ... CONFIG GET maxmemory-policy` → `allkeys-lru`
- [ ] `REDIS_PASSWORD` 미설정 시 컨테이너 부팅 실패 (`:?password 필수`)
- [ ] keyspace event 활성 (`CONFIG GET notify-keyspace-events` → `KEA`)
- [ ] **인증 없이 접속 시 거부**: `redis-cli -h localhost -p 6379 PING` → `NOAUTH` 에러 (REQ-NF-SEC-001 사내망 전제 위에 추가 가드)

### 회귀 시나리오 (T-I)
- **케이스 1 (정상 부팅)**: 패스워드 정상 → 10초 내 healthy + PING/PONG
- **케이스 2 (인증 실패)**: 잘못된 패스워드 → `WRONGPASS` 에러
- **케이스 3 (AOF 영속성)**: SET → down → up → GET 성공
- **케이스 4 (메모리 정책)**: 1 GB 채우기 시도 → LRU eviction 발생, `INFO stats evicted_keys` 증가

## :memo: Implementation Notes
- **기술 힌트**:
  - `redis:7-alpine`은 약 35 MB — 디스크 절약 (NFR-COS-001)
  - `notify-keyspace-events KEA`로 cache invalidation 자동 (Caffeine과 결합 시 `__keyevent@*__:expired` 구독)
  - DEV 환경에서 `bind 127.0.0.1`은 컨테이너 내부 — Docker network 통신 차단됨. `bind 0.0.0.0` + ports `127.0.0.1:6379:6379`가 옳음 (외부에서는 localhost로만)
- **유사 패턴**:
  - PostgreSQL (TK-00-1-1)과 동일 패턴 (healthcheck·volume·restart·`:?password 필수`)
  - NGINX (TK-00-1-3) 도 유사 (ports·healthcheck)
- **표준 참조**:
  - Redis Config Docs: https://redis.io/docs/management/config-file/
  - Redis Keyspace Notifications: https://redis.io/docs/manual/keyspace-notifications/
