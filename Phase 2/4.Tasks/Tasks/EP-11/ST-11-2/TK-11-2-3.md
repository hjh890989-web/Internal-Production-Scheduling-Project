### Flyway `V4_4_data__audit_admin_role.sql`

```sql
-- audit_admin: 파티션 DDL 권한 (Phase 2+ archive job용)
DO $$
BEGIN
    IF NOT EXISTS (SELECT 1 FROM pg_roles WHERE rolname = 'audit_admin') THEN
        CREATE ROLE audit_admin NOLOGIN;
    END IF;
END$$;

-- audit_admin: schema에 CREATE TABLE 권한 (파티션 추가용)
GRANT USAGE, CREATE ON SCHEMA audit TO audit_admin;
GRANT SELECT ON ALL TABLES IN SCHEMA audit TO audit_admin;

-- UPDATE/DELETE는 명시적 REVOKE (TK-11-2-1과 동일)
REVOKE UPDATE, DELETE, TRUNCATE ON ALL TABLES IN SCHEMA audit FROM audit_admin;
ALTER DEFAULT PRIVILEGES IN SCHEMA audit
    REVOKE UPDATE, DELETE, TRUNCATE ON TABLES FROM audit_admin;
ALTER DEFAULT PRIVILEGES IN SCHEMA audit
    GRANT SELECT ON TABLES TO audit_admin;

-- role 계층 명시
COMMENT ON ROLE audit_reader IS 'audit 조회 전용 (감사역·PM)';
COMMENT ON ROLE audit_admin  IS 'audit 파티션 관리 (DBA) — UPDATE/DELETE 여전히 불가';
```

### `AuditPartitionMaintenanceJob.java`

```java
@Component @RequiredArgsConstructor @Slf4j
public class AuditPartitionMaintenanceJob {

    private final JdbcTemplate jdbc;

    /**
     * 매월 1일 — 다음 달 + 6개월 분량 파티션 사전 생성.
     * 6개월 buffer로 운영팀 장애 대응 시간 확보.
     */
    @Scheduled(cron = "0 0 2 1 * *")    // 매월 1일 02:00
    public void createNextMonthPartitions() {
        var nextStart = LocalDate.now().plusMonths(6).withDayOfMonth(1);

        for (String table : List.of("vc_schedule_log", "ex_schedule_log", "order_log")) {
            createPartitionIfMissing(table, nextStart);
        }
    }

    private void createPartitionIfMissing(String parentTable, LocalDate partStart) {
        String partName = parentTable + "_" + partStart.format(DateTimeFormatter.ofPattern("yyyyMM"));
        String partEnd = partStart.plusMonths(1).toString();

        try {
            jdbc.execute(String.format(
                "CREATE TABLE IF NOT EXISTS audit.%s PARTITION OF audit.%s " +
                "FOR VALUES FROM ('%s') TO ('%s')",
                partName, parentTable, partStart, partEnd
            ));
            log.info("Audit partition created: {}", partName);
        } catch (Exception e) {
            log.error("Audit partition 생성 실패: {} — 운영팀 확인 필요", partName, e);
        }
    }
}
```

```markdown
# Audit 3년 보존 정책 (NFR-SEC-004)

## 보존 원칙
- audit row는 occurred_at 기준 3년 보관
- 보존 기간 내 변조 불가 (TK-11-2-1·11-2-2)

## 파티션 관리
- 월별 partition (audit.vc_schedule_log_YYYYMM)
- `AuditPartitionMaintenanceJob`이 매월 1일 6개월 buffer 생성
- 수동 추가: `audit_admin` role로 CREATE TABLE ... PARTITION OF

## 만료 archive (Phase 2+ 구현)
1. 3년 + 1개월 경과 partition 식별
2. `pg_dump` archive schema로 export
3. 외부 S3/Glacier 백업
4. 원본 partition은 그대로 (DROP 금지 — 변조 불가 원칙)

## 응급 archive (보존 기간 초과)
- 별도 PR + 보안팀·법무팀 승인 절차
- audit 자체의 변경은 별도 audit 추적 필요 (chain of custody)
```
