### Flyway `V4_4__audit_immutability.sql`

```sql
-- 1) audit.* 기존 테이블에서 UPDATE·DELETE·TRUNCATE 권한 제거
REVOKE UPDATE, DELETE, TRUNCATE ON ALL TABLES IN SCHEMA audit FROM PUBLIC;
REVOKE UPDATE, DELETE, TRUNCATE ON ALL TABLES IN SCHEMA audit FROM app_user;
REVOKE UPDATE, DELETE, TRUNCATE ON ALL TABLES IN SCHEMA audit FROM planner_role;
REVOKE UPDATE, DELETE, TRUNCATE ON ALL TABLES IN SCHEMA audit FROM master_admin;

-- 2) DEFAULT PRIVILEGES — 향후 audit 스키마에 추가될 테이블도 자동 적용
ALTER DEFAULT PRIVILEGES IN SCHEMA audit
    REVOKE UPDATE, DELETE, TRUNCATE ON TABLES FROM PUBLIC;
ALTER DEFAULT PRIVILEGES IN SCHEMA audit
    REVOKE UPDATE, DELETE, TRUNCATE ON TABLES FROM app_user;

-- 3) audit_reader role — SELECT 전용
DO $$
BEGIN
    IF NOT EXISTS (SELECT 1 FROM pg_roles WHERE rolname = 'audit_reader') THEN
        CREATE ROLE audit_reader NOLOGIN;
    END IF;
END$$;

GRANT USAGE ON SCHEMA audit TO audit_reader;
GRANT SELECT ON ALL TABLES IN SCHEMA audit TO audit_reader;
ALTER DEFAULT PRIVILEGES IN SCHEMA audit
    GRANT SELECT ON TABLES TO audit_reader;

-- 4) app_user는 INSERT (트리거가 사용) + SELECT만
GRANT INSERT, SELECT ON ALL TABLES IN SCHEMA audit TO app_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA audit
    GRANT INSERT, SELECT ON TABLES TO app_user;

-- 5) audit 스키마 자체 보호 — DROP 권한 superuser만
REVOKE CREATE ON SCHEMA audit FROM PUBLIC;

-- 운영 가이드 commit:
-- - audit 테이블 변경(스키마)은 별도 PR + DBA 승인
-- - 보존 기간 만료 row archival은 별도 절차 (현재 미구현)
```

### `docs/operations/audit_immutability_policy.md`

```markdown
# Audit 불변성 정책 (BR-X02 / NFR-SEC-004)

## 원칙
- audit 스키마 row는 INSERT 이후 변경 불가
- 모든 role (admin 포함) UPDATE/DELETE/TRUNCATE 권한 없음
- 보존: 3년 (NFR-SEC-004)

## 새 audit 테이블 추가 시
1. 마이그레이션에 트리거 함수 + AFTER INSERT/UPDATE/DELETE 트리거
2. PARTITION BY RANGE (occurred_at) — 월별
3. `ALTER DEFAULT PRIVILEGES`가 자동 REVOKE 적용

## 응급 archive (3년 보존 후)
- 별도 archive 스키마로 INSERT INTO ... SELECT
- 원본 row는 그대로 (자동 partition drop은 Phase 2+ 검토)

## 보안 사건 발생 시
- audit를 변경해야 한다면 = forensic 손상 = 즉시 보고
- DBA가 superuser로 변경 시도 시 pg_audit 로깅
```

## :shield: Risk Mitigation

- DEFAULT PRIVILEGES — 향후 신규 audit 테이블 생성 시 권한 누락 차단
- superuser 우회는 별도 정책 — 일반 운영에서는 불가능
- TRUNCATE는 별도 명시 REVOKE — DELETE보다 권한 강함 (CASCADE 위험)

## :memo: Implementation Notes

- PostgreSQL DEFAULT PRIVILEGES는 owner 기준 — Flyway가 master_admin이라면 master_admin이 생성하는 향후 audit 테이블에 자동 적용
- Phase 2+ 검토: `pgaudit` extension 또는 row-level append-only 기능
