#### A. `SlotCompatibilityMatrix.java` — 자료 구조

```java
package com.scheduling.master.vc;

import lombok.Value;

import java.time.Instant;
import java.util.*;

@Value
public class SlotCompatibilityMatrix {

    int version;                                          // monotonic
    Instant builtAt;
    Map<String, Map<SlotPosition, Boolean>> byHose;       // 47품번 × 7 슬롯
    Map<SlotPosition, Set<String>> bySlot;                // 슬롯 → 가능 품번 (역인덱스)
    Set<String> unschedulableHoseIds;                     // BR-V11

    public boolean isEligible(String hoseId, SlotPosition slot) {
        var m = byHose.get(hoseId);
        return m != null && Boolean.TRUE.equals(m.get(slot));
    }

    public Set<String> eligibleHoseIdsFor(SlotPosition slot) {
        return bySlot.getOrDefault(slot, Set.of());
    }

    public Set<SlotPosition> eligibleSlotsFor(String hoseId) {
        var m = byHose.get(hoseId);
        if (m == null) return Set.of();
        return m.entrySet().stream()
            .filter(Map.Entry::getValue)
            .map(Map.Entry::getKey)
            .collect(java.util.stream.Collectors.toUnmodifiableSet());
    }
}
```

#### B. `SlotCompatibilityMatrixService.java` — 빌드 + 캐시

```java
package com.scheduling.master.vc;

import com.github.benmanes.caffeine.cache.Caffeine;
import com.github.benmanes.caffeine.cache.Cache;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;

import jakarta.annotation.PostConstruct;
import java.time.Duration;
import java.time.Instant;
import java.util.*;
import java.util.concurrent.atomic.AtomicInteger;
import java.util.concurrent.atomic.AtomicReference;

@Slf4j
@Service
@RequiredArgsConstructor
public class SlotCompatibilityMatrixService {

    private final VcConstraintRepository repo;

    private final Cache<String, SlotCompatibilityMatrix> cache = Caffeine.newBuilder()
        .expireAfterWrite(Duration.ofHours(1))
        .maximumSize(1)                                   // 단일 매트릭스만 보관
        .build();

    private final AtomicReference<SlotCompatibilityMatrix> current = new AtomicReference<>();
    private final AtomicInteger versionCounter = new AtomicInteger(0);

    @PostConstruct
    public void initialBuild() {
        rebuild();
    }

    /**
     * 빌드 — 전체 VC_CONSTRAINT 조회 후 매트릭스 생성.
     * TC-VC-001: ≤ 1초 (47품번 기준 평균 50ms)
     */
    public synchronized SlotCompatibilityMatrix rebuild() {
        long start = System.nanoTime();
        var all = repo.findAll();

        var byHose = new HashMap<String, Map<SlotPosition, Boolean>>();
        var bySlot = new EnumMap<SlotPosition, Set<String>>(SlotPosition.class);
        var unschedulable = new HashSet<String>();

        for (SlotPosition s : SlotPosition.values()) bySlot.put(s, new HashSet<>());

        for (VcConstraint c : all) {
            var hoseSlots = new EnumMap<SlotPosition, Boolean>(SlotPosition.class);
            for (SlotPosition s : SlotPosition.values()) {
                boolean ok = c.isEligibleFor(s);
                hoseSlots.put(s, ok);
                if (ok) bySlot.get(s).add(c.getHoseId());
            }
            byHose.put(c.getHoseId(), Map.copyOf(hoseSlots));

            if (c.isUnschedulable()) unschedulable.add(c.getHoseId());
        }

        var matrix = new SlotCompatibilityMatrix(
            versionCounter.incrementAndGet(),
            Instant.now(),
            Map.copyOf(byHose),
            Collections.unmodifiableMap(bySlot.entrySet().stream()
                .collect(java.util.stream.Collectors.toMap(
                    Map.Entry::getKey,
                    e -> Set.copyOf(e.getValue())))),
            Set.copyOf(unschedulable)
        );

        current.set(matrix);
        cache.put("default", matrix);

        long elapsedMs = (System.nanoTime() - start) / 1_000_000;
        log.info("Slot matrix rebuilt: v{}, {} hoses, {} unschedulable, elapsed={}ms",
            matrix.version(), all.size(), unschedulable.size(), elapsedMs);

        if (elapsedMs > 1000) {
            log.warn("Slot matrix rebuild SLA violation: {}ms (limit 1000ms)", elapsedMs);
        }

        return matrix;
    }

    public SlotCompatibilityMatrix current() {
        return current.get();
    }

    public synchronized void invalidate() {
        log.info("Slot matrix cache invalidated — will rebuild on next access");
        cache.invalidateAll();
        rebuild();
    }
}
```

#### C. `VcConstraintChangeListener.java` — PG LISTEN

```java
package com.scheduling.master.vc;

import com.impossibl.postgres.api.jdbc.PGConnection;
import com.impossibl.postgres.api.jdbc.PGNotificationListener;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.boot.context.event.ApplicationReadyEvent;
import org.springframework.context.event.EventListener;
import org.springframework.stereotype.Component;

import javax.sql.DataSource;
import java.sql.Connection;
import java.sql.Statement;

@Slf4j
@Component
@RequiredArgsConstructor
public class VcConstraintChangeListener implements PGNotificationListener {

    private final DataSource dataSource;
    private final SlotCompatibilityMatrixService matrixService;

    @EventListener(ApplicationReadyEvent.class)
    public void start() throws Exception {
        Connection conn = dataSource.getConnection();
        PGConnection pgConn = conn.unwrap(PGConnection.class);
        pgConn.addNotificationListener(this);

        try (Statement st = conn.createStatement()) {
            st.execute("LISTEN vc_constraint_changed");
            log.info("Subscribed to PG channel 'vc_constraint_changed'");
        }
    }

    @Override
    public void notification(int processId, String channelName, String payload) {
        log.info("VC_CONSTRAINT changed (hoseId={}) — invalidating matrix", payload);
        matrixService.invalidate();
    }
}
```

## :memo: Implementation Notes

- Caffeine vs ConcurrentHashMap: 본 케이스는 단일 매트릭스이지만 Caffeine으로 통일 (다른 캐시와 정합)
- `Map.copyOf` + `Set.copyOf`로 불변성 강제 — JDK 10+
- LISTEN/NOTIFY는 PG-specific — 다른 DBMS 전환 시 Redis Pub/Sub로 대체 (ADR-011)
- `@PostConstruct`는 Bean lifecycle 시작 시점 — `ApplicationReadyEvent`로 늦추는 방안도 있음 (Repository ready 보장)
