#### B. `DuplicateOrderException.java`

```java
package com.scheduling.order.domain;

import lombok.Getter;
import java.time.LocalDate;

@Getter
public class DuplicateOrderException extends RuntimeException {
    private final String hoseId;
    private final LocalDate deliveryDate;
    private final int masterVersion;

    public DuplicateOrderException(String hoseId, LocalDate deliveryDate, int masterVersion, Throwable cause) {
        super("중복 수주: 품번=%s, 납기=%s, 버전=%d (BR-OC-005)".formatted(hoseId, deliveryDate, masterVersion), cause);
        this.hoseId = hoseId;
        this.deliveryDate = deliveryDate;
        this.masterVersion = masterVersion;
    }
}
```

#### C. `OrderRepository`에서 DB 예외 → 도메인 예외 변환

```java
package com.scheduling.order.domain;

import org.springframework.dao.DataIntegrityViolationException;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

@Repository
public interface OrderRepository extends JpaRepository<Order, UUID> {
    // 표준 JPA 메서드만 정의, 변환은 서비스 레이어에서
}

// 서비스 레이어 예시
@Service
public class OrderCommitService {
    private final OrderRepository repo;

    public Order commit(OrderDraft draft) {
        try {
            return repo.save(toEntity(draft));
        } catch (DataIntegrityViolationException e) {
            // PostgreSQL constraint name으로 판별
            if (e.getMessage() != null && e.getMessage().contains("uq_order_hose_delivery_version")) {
                throw new DuplicateOrderException(
                    draft.hoseId(), draft.deliveryDate(), draft.masterVersion(), e);
            }
            throw e;
        }
    }
}
```

#### D. `OrderExceptionHandler.java`

```java
package com.scheduling.order.api;

import com.scheduling.order.domain.DuplicateOrderException;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@RestControllerAdvice
public class OrderExceptionHandler {

    @ExceptionHandler(DuplicateOrderException.class)
    public ResponseEntity<DuplicateOrderResponse> handleDuplicate(DuplicateOrderException e) {
        return ResponseEntity.status(HttpStatus.CONFLICT).body(
            new DuplicateOrderResponse(
                "DUPLICATE_ORDER",
                e.getMessage(),
                e.getHoseId(),
                e.getDeliveryDate().toString(),
                e.getMasterVersion()
            )
        );
    }
}

public record DuplicateOrderResponse(
    String errorCode,
    String message,         // 한국어
    String hoseId,
    String deliveryDate,
    int masterVersion
) {}
```

## :shield: Risk Mitigation

- 잔여: PostgreSQL constraint name이 메시지에 등장하지 않는 드라이버 버전 → JPA `@UniqueConstraint` 어노테이션으로 ConstraintName 명시 (HikariCP·PgJDBC 4.x+ 정상)
- 동시성: 같은 키 동시 INSERT 시 한 쪽은 SQLState 23505 — 우리 핸들러가 정확히 catch

## :memo: Implementation Notes

- Spring Data JPA `save()`는 lazy flush — `saveAndFlush()`로 즉시 violation 검출 권장
- 부분 인덱스 (`WHERE status='ACTIVE'`) 옵션은 soft-delete 정책 도입 시 추가
- ConstraintName으로 식별이 안정적 (메시지 텍스트 의존 X) — JPA 어노테이션 `@Table(uniqueConstraints = @UniqueConstraint(name="uq_order_hose_delivery_version", ...))`
