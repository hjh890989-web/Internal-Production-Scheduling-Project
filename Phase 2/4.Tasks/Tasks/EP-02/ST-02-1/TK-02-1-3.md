public record OrderKey(String hoseId, LocalDate deliveryDate) {
    public static OrderKey of(OrderDraft draft) {

#### B. `DuplicateGroup.java`

```java
package com.scheduling.order.domain;

import java.util.List;

public record DuplicateGroup(
    OrderKey key,
    List<OrderDraft> candidates,    // 신규 batch 내 후보
    Order existingMaster            // 기존 마스터 row (있으면, 없으면 null)
) {
    public boolean hasExisting() { return existingMaster != null; }
    public int candidateCount() { return candidates.size(); }
    public boolean isDuplicate() {
        return candidates.size() > 1 || existingMaster != null;
    }
}
```

#### C. `DuplicateDetectionService.java` — 핵심

```java
package com.scheduling.order.domain;

import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

import java.util.*;
import java.util.stream.Collectors;

@Service
@RequiredArgsConstructor
public class DuplicateDetectionService {

    private final OrderRepository repo;

    /**
     * batch 단위 중복 사전 감지.
     *
     * @return DuplicateGroup 리스트 — isDuplicate()=true 인 그룹만 추출
     */
    public List<DuplicateGroup> detect(List<OrderDraft> batch) {
        // 1. batch 내 그룹핑 (OrderKey 기준)
        Map<OrderKey, List<OrderDraft>> grouped = batch.stream()
            .collect(Collectors.groupingBy(OrderKey::of));

        // 2. 기존 마스터 한 번에 조회 (N+1 방지)
        Set<OrderKey> keys = grouped.keySet();
        Map<OrderKey, Order> existing = repo.findActiveByKeys(keys).stream()
            .collect(Collectors.toMap(
                o -> new OrderKey(o.getHoseId(), o.getDeliveryDate()),
                o -> o));

        // 3. DuplicateGroup 생성 (중복 있는 그룹만)
        return grouped.entrySet().stream()
            .map(e -> new DuplicateGroup(
                e.getKey(),
                e.getValue(),
                existing.get(e.getKey())))
            .filter(DuplicateGroup::isDuplicate)
            .toList();
    }

    /**
     * batch에서 중복 키만 추출 (간단 사용).
     */
    public Set<OrderKey> findDuplicateKeys(List<OrderDraft> batch) {
        return detect(batch).stream()
            .map(DuplicateGroup::key)
            .collect(Collectors.toSet());
    }
}
```

#### D. `OrderRepository`에 batch 조회 메서드 추가

```java
public interface OrderRepository extends JpaRepository<Order, UUID> {

    @Query("""
        SELECT o FROM Order o
        WHERE o.status = 'ACTIVE'
          AND (o.hoseId, o.deliveryDate) IN :keys
        """)
    List<Order> findActiveByKeys(@Param("keys") Set<OrderKey> keys);
}
```

> **참고**: PostgreSQL의 row-value `IN` 구문 지원. Spring Data JPA에서 직접 지원 안 되면 native query로 전환:
> ```java
> @Query(value = "SELECT * FROM app.order WHERE status='ACTIVE' AND (hose_id, delivery_date) IN (:keys)",
>        nativeQuery = true)
> ```

## :test_tube: Acceptance Criteria

**검증**: T-U + I

- [ ] **batch 내 중복**: 같은 OrderKey 2개 OrderDraft → DuplicateGroup 1개 (candidates 2)
- [ ] **기존 마스터와 중복**: batch 1개 + DB 기존 1개 → DuplicateGroup 1개 (candidates 1, existing not null)
- [ ] **양쪽 모두 중복**: batch 2개 + DB 기존 → DuplicateGroup 1개 (candidates 2, existing not null)
- [ ] **중복 없음**: 모두 유니크 → 빈 리스트
- [ ] **성능**: 10,000 row batch + 100,000 row 마스터 → ≤2초 (N+1 회피, batch query)
- [ ] **빈 batch**: 빈 리스트 반환 (예외 없음)
- [ ] **단위 테스트 ≥ 8 케이스**
