### `docs/operations/idp-failover.md`

```markdown
# 사내 IdP 장애 시 Fallback 절차

## Trigger 조건
- 사내 IdP (AD FS·Entra ID) 미응답 5초 timeout
- SAML SSO redirect 실패
- 사용자가 IT에 로그인 불가 보고

## Fallback 흐름
1. Keycloak 로그인 페이지에서 "사내 SSO" 시도
2. 5초 timeout 시 "ID/PW 직접 로그인" 옵션 자동 표시
3. Local user (IT_OPS가 사전 생성한 emergency 계정) 로그인
4. 비밀번호 정책 동일 (12자/3종/5회 잠금)

## Emergency 계정 사전 준비
- IT_OPS·Planner role 별 1~2개 emergency 계정
- 비밀번호는 봉인 보관 (사고 시점에 IT lead 결재 후 사용)
- 사용 후 즉시 비밀번호 변경 + audit 보고
```
