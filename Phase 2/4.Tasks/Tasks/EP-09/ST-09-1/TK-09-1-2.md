### 핵심 로직

#### A. `SettingGroupAllocator.java`

```java
package com.scheduling.ex.allocator;

import com.scheduling.ex.master.setting.SettingGroupCache;
import com.scheduling.ex.master.shift.ShiftEfficiencyService;
import com.scheduling.ex.schedule.ExScheduleCandidate;
import com.scheduling.ex.yield.YieldFormula;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

import java.time.LocalDate;
import java.util.*;

@Service @RequiredArgsConstructor
public class SettingGroupAllocator {

    private static final List<String> SHIFT_CODES = List.of(
        "DAY_FIRST", "DAY_SECOND", "NIGHT_FIRST", "NIGHT_SECOND");

    private final SettingGroupCache settingGroupCache;
    private final ShiftEfficiencyService shiftService;
    private final YieldFormula yieldFormula;
    private final ExConstraintRepository exRepo;
    private final MachineRepository machineRepo;

    /**
     * BR-E06·E07: 후보를 (date, machine, shift) 슬롯에 그룹 단위 배치.
     *
     * 알고리즘:
     *   1. 후보를 setting_group별 분류
     *   2. 각 (date, machine, shift) 슬롯에 대해 가장 많은 yield를 생산 가능한 그룹 선택
     *   3. 같은 그룹 품번들 모아서 단일 shift slot에 배치 (BR-E06 셋업 0)
     *   4. yield 합산 ≤ effective_min × speed_max 제약 강제
     *   5. 잔여 후보는 다음 슬롯 (다른 shift 또는 다른 머신·다른 날)
     */
    public ExAllocationResult allocate(ExAllocationContext ctx) {
        var schedule = new ArrayList<ExAllocationRow>();
        var conflicts = new ArrayList<ExAllocationConflict>();
        var remaining = new ArrayDeque<>(ctx.candidates());

        for (LocalDate date : ctx.workingDays()) {
            for (var machine : machineRepo.findExMachines()) {
                for (String shiftCode : SHIFT_CODES) {
                    if (remaining.isEmpty()) break;

                    var slot = new ShiftSlot(date, machine.id(), shiftCode);
                    var assigned = assignShiftSlot(slot, remaining, ctx);
                    schedule.addAll(assigned);
                }
            }
        }

        // 잔여 후보 = capa 부족 → conflict
        for (var unassigned : remaining) {
            conflicts.add(ExAllocationConflict.of(unassigned,
                "BR-E06·E07: 4주 호라이즌 내 셋팅 그룹 배치 불가"));
        }

        return new ExAllocationResult(schedule, conflicts);
    }

    private List<ExAllocationRow> assignShiftSlot(ShiftSlot slot,
                                                    Deque<ExScheduleCandidate> remaining,
                                                    ExAllocationContext ctx) {
        // 1) 셋팅 그룹별 후보 분류
        Map<Integer, List<ExScheduleCandidate>> byGroup = new HashMap<>();
        for (var cand : remaining) {
            int group = settingGroupCache.primaryGroup(cand.hoseId());
            byGroup.computeIfAbsent(group, k -> new ArrayList<>()).add(cand);
        }

        // 2) 가장 yield 합산 큰 그룹 선택 (BR-E07 동시 생산 최대화)
        int effectiveMin = shiftService.effectiveMin(slot.shiftCode());
        int selectedGroup = byGroup.entrySet().stream()
            .max(Comparator.comparingLong(e -> estimateYield(e.getValue(), effectiveMin)))
            .map(Map.Entry::getKey)
            .orElse(-1);

        if (selectedGroup == -1) return List.of();

        // 3) 선택 그룹의 후보들을 시간 누적해서 배치
        var assigned = new ArrayList<ExAllocationRow>();
        int remainingMin = effectiveMin;

        var groupCandidates = byGroup.get(selectedGroup).iterator();
        while (groupCandidates.hasNext() && remainingMin > 0) {
            var cand = groupCandidates.next();
            var exMaster = exRepo.findById(cand.hoseId()).orElseThrow();

            // 후보당 필요 시간 = qExt × lengthMm / (speed × 1000)
            int neededMin = (int) Math.ceil(
                cand.getQExt().doubleValue() * exMaster.getLengthMm() /
                (exMaster.getSpeedMPerMin().value().doubleValue() * 1000));

            int actualMin = Math.min(neededMin, remainingMin);
            long actualYield = yieldFormula.compute(
                exMaster.getSpeedMPerMin(), actualMin, exMaster.getLength());

            assigned.add(ExAllocationRow.of(slot, cand, selectedGroup, actualMin, actualYield));
            remainingMin -= actualMin;

            if (actualYield >= cand.getQExt()) {
                remaining.remove(cand);     // 충족 — 후보 제거
            } else {
                cand.subtractQExt(actualYield);    // 부분 배치 — 잔여 유지
            }
        }

        return assigned;
    }

    private long estimateYield(List<ExScheduleCandidate> group, int effectiveMin) {
        // 단순화: 그룹 yield 합산 추정 — 실 마스터 조회로 정교화 가능
        return group.stream()
            .mapToLong(c -> Math.min(c.getQExt(),
                yieldFormula.compute(c.getSpeed(), effectiveMin, c.getLength())))
            .sum();
    }
}
```

## :shield: Risk Mitigation

- Greedy 단순화 — 최적 yield 보장 못함, 그러나 셋업 0건은 보장 (Sprint 3 DoD 핵심)
- 향후 LP solver 또는 simulated annealing 검토 (Phase 2+)
- 다중 그룹 품번의 secondary 그룹 사용 — primary 그룹 capa 부족 시 fallback (현재 미구현, 향후)

## :memo: Implementation Notes

- `estimateYield`는 휴리스틱 — 최적해 보장 아님, 셋업 0 + 합리적 yield 균형
- 동일 그룹 내 품번 순서는 `qExt` 큰 순 (납기 압박 우선) — 추후 우선순위 마스터 도입 가능
- conflict는 EP-EX12 (충돌 대안)에서 ≥ 3 distinct 대안 제시
