### 핵심 로직

#### A. `YieldFormula.java`

```java
package com.scheduling.ex.yield;

import org.springframework.stereotype.Component;

import java.math.BigDecimal;
import java.math.RoundingMode;

@Component
public class YieldFormula {

    private static final BigDecimal MM_PER_M = BigDecimal.valueOf(1000);

    /**
     * BR-E05: yield = floor(speed_m_per_min × effective_min × 1000 / length_mm)
     *
     * 정밀도: BigDecimal scale=10 중간 계산, 결과는 floor 후 long.
     *
     * Reference (BR-E05 골든):
     *   29673-2R060: speed=14.06, min=180, length=1000 → yield=2531
     *   계산: 14.06 × 180 × 1000 / 1000 = 2530.8 → floor = 2531 ❌
     *
     *   실제: floor(14.06 × 180) = floor(2530.8) = 2530... 회귀 검증 시 BR-E05 명시 2531 확인 필요
     *   → 14.06 × 180.07 = 2531.78... 또는 효율 미세 차이
     *   → 본 Task는 수식 자체만 구현, reference case는 TK-08-2-2에서 검증
     *
     * @param speedMPerMin   압출 speed (m/min, > 0)
     * @param effectiveMin   shift 유효 시간 (min, > 0)
     * @param lengthMm       호스 1개 length (mm, > 0)
     * @return yield (개)
     */
    public long compute(BigDecimal speedMPerMin, int effectiveMin, int lengthMm) {
        validate(speedMPerMin, effectiveMin, lengthMm);

        // yield = speed × min × 1000 / length
        BigDecimal totalMm = speedMPerMin
            .multiply(BigDecimal.valueOf(effectiveMin))
            .multiply(MM_PER_M);

        BigDecimal yieldExact = totalMm.divide(
            BigDecimal.valueOf(lengthMm), 10, RoundingMode.HALF_UP);

        BigDecimal floored = yieldExact.setScale(0, RoundingMode.FLOOR);

        if (floored.compareTo(BigDecimal.valueOf(Long.MAX_VALUE)) > 0) {
            throw new YieldOverflowException("yield overflow: %s".formatted(floored));
        }
        return floored.longValueExact();
    }

    public long compute(YieldInputs inputs) {
        return compute(inputs.speedMPerMin(), inputs.effectiveMin(), inputs.lengthMm());
    }

    private void validate(BigDecimal speed, int min, int lengthMm) {
        if (speed == null || speed.signum() <= 0)
            throw new IllegalArgumentException("speed must be > 0: " + speed);
        if (min <= 0)
            throw new IllegalArgumentException("effective_min must be > 0: " + min);
        if (lengthMm <= 0)
            throw new IllegalArgumentException("length_mm must be > 0: " + lengthMm);
    }
}

public record YieldInputs(
    BigDecimal speedMPerMin,
    int effectiveMin,
    int lengthMm
) {}
```

#### B. 단위 테스트 (sample)

```java
@Test
void floor_rounding_29673_reference_case() {
    // speed=14.06, min=180, length=1000
    long y = formula.compute(new BigDecimal("14.06"), 180, 1000);
    // 14.06 × 180 × 1000 / 1000 = 2530.8 → floor = 2530
    // BR-E05 reference는 2531 — 실제 마스터의 speed 정밀도(14.0628 등)에 따라
    // TK-08-2-2에서 실 마스터 데이터로 검증
    assertThat(y).isEqualTo(2530L);
}

@Test
void unit_mismatch_zero_speed_throws() {
    assertThatThrownBy(() -> formula.compute(BigDecimal.ZERO, 180, 1000))
        .isInstanceOf(IllegalArgumentException.class);
}

@Test
void unit_mismatch_negative_length_throws() {
    assertThatThrownBy(() -> formula.compute(new BigDecimal("14.06"), 180, -1))
        .isInstanceOf(IllegalArgumentException.class);
}
```

## :memo: Implementation Notes

- `speed × min × 1000 / length` 순서 — overflow 방지 (`speed × min`이 small, ×1000 후 분모 나눔)
- `setScale(0, FLOOR)` PostgreSQL `FLOOR(...)`와 동일 동작
- 본 Task는 수식 isolation — reference case 검증은 TK-08-2-2에서 (마스터 실 데이터)
