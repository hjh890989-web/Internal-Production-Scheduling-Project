## 3. 핵심 발견 및 MVP 반영 권고

### 3.1 DOS 기준 우선순위 (상위 4개 = Phase 1 필수)

```
1위. 수주 취합 자동화 + 변경 감지     (DOS=4.0) → M1 모듈 핵심
2위. 성형→압출 자동 역산 연동          (DOS=4.0) → M3 모듈 핵심  
3위. Key Person 리스크 해소 (제약 검증) (DOS=4.0) → M2 모듈 핵심
4위. 앵글 교체 최소화 배치 로직         (DOS=2.0) → M2 최적화 기능
```

### 3.2 숨겨진 Job (인터뷰에서 새로 발견)

| 발견 | 인터뷰이 | MVP 반영 |
|------|---------|---------|
| **엑셀 Export 필수** — 기존 수식 자산 활용 니즈 | P1 김정훈 | 모든 뷰에 엑셀 Export 기본 제공 |
| **책임 소재 명확화** — "시스템 제안 vs 사용자 확정" 구분 | P4 최민혁 | 자동 제안 후 "확정" 버튼 필수 |
| **과거 실패 트라우마** — 3년 전 바코드 시스템 폐기 경험 | P11 강병철 | 파일럿 1개월 후 KPI 비교 보고 필수 |

### 3.3 4 Forces 종합 매트릭스

| Force | 핵심 내용 | 대응 전략 |
|-------|----------|----------|
| **Push** | 수작업 취합 4시간, 변경 누락, 관체 부족, 금형 충돌 | → 자동 Import + 변경 감지 + 제약 검증 |
| **Pull** | 버튼 하나로 통합, 변경분 색상 표시, 자동 역산 알림 | → 핵심 UX: 원클릭 Import, 변경 하이라이트 |
| **Habit** | 7년 엑셀 수식, 15년 현장 경험, 종이 메모 루틴 | → 엑셀 Export 필수, 수동 드래그 조정 허용 |
| **Anxiety** | 시스템 신뢰도, 책임 소재, 과거 실패, IT 문해력 | → "제안+확정" 분리, 대형 글씨, 자동 로그인 |
