## 1. 문제 정의 — 쉬운 말로

### 1.1 한 문장으로

> **매주 엑셀 3개를 손으로 합치느라 반나절을 쓰고, 고객이 수량 바꾸면 또 다시 하고,
> 이 일을 할 줄 아는 사람이 한 명뿐이라 그 사람 연차 쓰면 공장이 흔들리는 상황입니다.**

### 1.2 좋은 문제 정의서 3가지 조건 체크

| 조건 | 원본 점수 | 코멘트 |
|------|:--------:|--------|
| ① 어떤 사람의 문제인가 | ⚠️ 50점 | "생산관리 담당자 3~5명"이라고만 되어 있음. 김정훈 주임이 월요일 아침에 얼마나 바쁜지가 안 보임 |
| ② 충분히 넓은가 | ❌ 30점 | Import 엔진, 간트차트까지 확정해서 다른 해결책을 생각할 여지가 없음 |
| ③ MVP로 해결 가능한가 | ⚠️ 60점 | 범위는 있는데, 기술 스펙이랑 섞여서 뭐가 MVP이고 뭐가 아닌지 헷갈림 |

## 2. 이 문제를 겪는 사람들 — 실무자 언어로

### 2.1 김정훈 주임 (생산관리, 7년차)

**월요일 아침 루틴:**
1. 메일함에 엑셀 3개 도착 (월별 예상, KD 발주, 주간 발주)
2. 3개 다 포맷이 다름 → VLOOKUP + 복붙으로 통합본 만들기
3. **4.2시간** 걸림
4. 화요일에 고객사가 "수량 바꿨어요" 연락 → 다시 처음부터

**실제로 한 말:**
> *"지난달에 KD 발주 변경을 못 봐서 300개를 못 맞췄어요. 고객사에서 전화 왔을 때 식은땀이 났습니다."*

**실무자 번역:** 이 사람이 연차 쓰면 스케줄 짤 수 있는 사람이 없습니다. 7년 동안 쌓은 노하우가 전부 머릿속에 있어요. 엑셀 수식도 본인만 이해하는 구조고요.

---

### 2.2 최민혁 대리 (생산관리, 3년차)

**김정훈 주임 연차 날 상황:**
1. 긴급 수주 변경 들어옴
2. 금형 조건을 모름 (어떤 제품이 어떤 슬롯에 들어가는지)
3. "가장 안전한 방법" = 지난주 스케줄 복사해서 살짝 수정

**실제로 한 말:**
> *"지난달 정훈 주임 연차 때 IC 가류기에 저압 전용 제품을 넣어서 반장님한테 혼났어요."*

**실무자 번역:** 경험 없이는 실수가 나올 수밖에 없는 구조입니다. 금형·앵글·슬롯 조건이 너무 많아서 외울 수가 없어요. 근데 이걸 확인할 시스템이 없으니까 사람한테 전화하는 수밖에요.

---

### 2.3 박도영 반장 (압출 현장, 10년차)

**매일 반복되는 상황:**
1. 성형팀에서 스케줄 변경함
2. 카톡으로 알려주기로 했는데, 가끔 까먹음
3. 다음날 아침에야 "어젯밤에 엉뚱한 걸 뽑았네" 발견
4. 성형 라인 정지 → 관체(반제품) 부족

**실제로 한 말:**
> *"이번 달만 관체 부족이 3번 있었어요. 매번 성형 쪽 변경 때문이었습니다."*

**실무자 번역:** 성형이랑 압출이 같은 공장인데, 스케줄이 카톡으로 연결되어 있어요. 시스템 연결이 아니라 **사람 연결**입니다. 사람이 까먹으면 끝이에요.

---

### 2.4 이수진 반장 (성형 현장, 15년차)

**매주 반복되는 상황:**
1. 사무실에서 주간 스케줄 받음
2. 앵글 교체가 하루 6번으로 잡혀 있음
3. "이거 순서만 바꾸면 2번이면 되는데..." → 종이에 다시 짬

**실제로 한 말:**
> *"어제도 앵글을 5번 바꿨는데, 사실 순서만 바꾸면 2번이면 됐거든요."*

**실무자 번역:** 사무실 스케줄이 현장 사정을 반영 못 해서, 현장 반장이 매번 다시 짭니다. 사실상 사무실 스케줄이 **1차 초안**이고, 현장 반장의 경험이 **최종본**인 거예요.

## 3. 문제 구조 — 복잡하게 말하면 안 되는 이유

### 세 줄 요약

```
1. 엑셀이 3개인데 포맷이 다 달라서, 합치는 데 반나절 걸림
2. 스케줄을 짤 줄 아는 사람이 한 명이라, 그 사람 없으면 사고남
3. 성형이랑 압출이 카톡으로 연결되어서, 누락되면 라인 섬
```

### 좀 더 정리하면

| 문제 | 쉽게 말하면 | 피해 | 얼마나 자주 |
|------|-----------|------|-----------|
| **수주 데이터 파편화** | 엑셀 3개를 손으로 합침 | 4.2시간/주 낭비 + 누락·중복 | 매주 |
| **수주 변경 추적 불가** | 고객이 바꾸면 카톡으로 옴, 놓치면 끝 | 납기 지연 → 고객 클레임 | 거의 매일 |
| **Key Person 의존** | 김정훈 주임 없으면 스케줄 못 짬 | 제약 위반 → 현장 사고 | 연차/출장 시마다 |
| **공정 간 단절** | 성형↔압출이 카톡·전화로 연결 | 관체 부족 → 라인 정지 (월 3건) | 변경 시마다 |
| **스케줄 현장 괴리** | 사무실 스케줄을 현장이 다시 짬 | 앵글 교체 과다 → 생산 손실 | 매주 |

## 4. MVP 범위 — 내일 당장 뭘 해야 하나

### 4.1 Phase 1에서 풀 문제 (3개만)

| # | 문제 | 왜 먼저? | 누가 좋아하나 |
|---|------|---------|-------------|
| P1 | 엑셀 3개 합치기 + 변경 추적 | GAP 4 (가장 아픈 곳) | 김정훈, 최민혁 |
| P2 | 금형·슬롯 제약 자동 확인 | GAP 4 (사고 방지) | 최민혁, 이수진 |
| P3 | 성형↔압출 스케줄 연동 | GAP 4 (라인 정지 방지) | 박도영 |

### 4.2 Phase 1에서 안 하는 것 (명확히)

| 안 하는 것 | 왜 안 하나 |
|-----------|-----------|
| 자재 발주(MRP) | 다른 영역임. 나중에 해도 됨 |
| 경영진 대시보드 | 데이터가 쌓여야 의미 있음 |
| 모바일 야간 조회 | 중요하지만 Phase 2 |
| 품질 분석 연계 | 데이터 축적 먼저 |

### 4.3 적용 범위

| 항목 | 내용 |
|------|------|
| 제품 | 전체 X → **파일럿 제품군만** 먼저 |
| 공정 | 수주 → 성형 → 압출 |
| 사용자 | 약 20명 |
| 서버 | 사내 온프레미스 |

## 5. 현재 숫자 (이게 있어야 목표를 말할 수 있습니다)

> [!IMPORTANT]
> **"80% 단축"이라고 쓰려면, "지금 몇 시간인데요?"라는 질문에 답할 수 있어야 합니다.**
> 아래는 JTBD 인터뷰에서 나온 실제 숫자입니다.

| 뭘 측정하나 | 지금 수치 | 어디서 나온 건가 | 추가 확인 |
|-----------|----------|---------------|:--------:|
| 주간 엑셀 합치기 시간 | **4.2시간** | 김정훈 자체 측정 | ⬜ 2주 더 |
| 월간 변경 누락 | **≥1건** (300개 납기 지연) | 김정훈 증언 | ⬜ 집계 |
| 월간 관체 부족 | **3건** | 박도영 증언 | ⬜ MES 확인 |
| 변경→압출 반영 | **~24시간** 지연 | 박도영 증언 | ⬜ |
| 앵글 교체/일 | **5~6회** (최적 시 2~3회) | 이수진 증언 | ⬜ 1주 관찰 |
